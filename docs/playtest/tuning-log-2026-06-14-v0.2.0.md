# v0.2.0 バランス再設計ログ (2026-06-14)

プレイヤーフィードバックへの対応。前バージョン(0.1.x)のシムは「生存時間」を最適化していたが、
ボットが100〜300万兵まで膨れて敵をワンショットしていた = 「数は多いが弱い」状態を再現していただけだった。
今回はシムに脅威メトリクス(敵の寿命・画面占有・進化/特殊発生数)を追加し、測る対象を是正した上で再調整。

## フィードバックと対応
1. **撃つと成長が途中からx2に進化できない** → 進化を「射撃回数」基準に。`evolveNeed = (滞空中に撃てる射撃数 × 0.42)`、
   兵数依存項(`soldiers/40`)を撤廃。斉射(volley)単位でカウントするので兵力非依存=巨大軍でも進化可能。
   (シム検証: evolveEvents ~20/run。前は射撃数が兵数比例で実質不可能だった)
2. **両方青でも片方が敵の利得になる** → 捨てゲートの敵強化(skipEffect/pendingMod)を完全撤廃。
   除染して良ゲートにした努力が敵強化で相殺される本末転倒を解消。
3. **敵が弱い(死線含め全体的に)** → 敵HPを「主弾N発で落ちる量」に兵力追従させTTKを一定化。
   `enemyBaseHp = max(lvFloor, bulletsToKill × mainBulletDmg(trackedPower)) × timeRamp × hardHpMul`。
   旧式(指数0.6・上限3倍)では巨大軍がワンショット → 是正。敵寿命 0.16s → ~2.2s。
4. **両方罠はやめて** → ゲートは必ず1つ以上良ゲートを保証(全部罠は出さない)。
5. **ゲート数を増やして複雑化** → `left/right`の2択固定を `cells[]` 配列化。序盤2、WAVE3+で3、死線で時々4択。
6. **特殊ゲート追加** → 連射⚡/拡散✷/シールド🛡/殲滅💥/精鋭★ の5種(`SPECIAL_INFO`)。良ゲートの16%。
7. **バージョン表示** → `const VERSION` をトップ画面とHUD左下に。CLAUDE.mdにpush時バンプ規則を明記。
8. **死亡後復活の削除**(前回): 亡霊隊長を撤廃済み。

## 調整中に発見・修正した問題 (シム駆動)
- **特殊ゲートが一切湧かないバグ**: TUNE書き換え時に `specialChance` を消していた(`Math.random() < undefined` = 常にfalse)。復活。
- **序盤即死**: lvFloor高 + 接触ダメージ兵力比例で10兵スタートが1接触半壊 → lvFloorを `lv*3+1` に低減、
  接触/被弾ダメージに「現兵力の25%/20%上限」を設定。
- **EMA死のスパイラル**: 兵数が1500万に膨張 → 敵HPが追従 → ÷2等で兵数暴落 → 13秒ラグで敵HPは過大なまま → 小軍で倒せず洪水死。
  - 対策(a) **兵数ソフトキャップ**: `gateResult` で兵数3000超のx2は線形加算に減衰(数値インフレと暴落を防止)。1500万→~1.7万。
  - 対策(b) **非対称EMA**: 上昇は遅く(13s, x2無双窓を保証)、下降は速く(3s, 暴落後すぐ敵弱体化)。
- **湧きの洪水**: 撃破レート≒75/bulletsToKill は兵力非依存。湧きを兵力スケールさせると巨大軍で必ず溢れる →
  湧きは時間(WAVE)と死線でのみスケール。自軍の弾は画面幅480pxのうち~110pxしかカバーできないため湧きレートを抑制。
- **シューター狙撃過多**: 死線で20体出て弾幕地獄 → 同時7体に制限。
- **死線は「数」でなく「質」で**: countGrow↓ / hpRamp↑(+10%/wave) / eliteChance↑(20%) に振り、エリート(HP×2.5・速い)主体の死線に。

## 既知の限界 / 今後
- スキル差が小さい(全員死線到達・~245s死亡)。ボットが粗い proxy なため。人間ではもっと幅が出る想定。
- 死線の死は決定的(~245s)。スキルで死線をどれだけ延命できるかが腕の見せ所だが、ボットでは差が出にくい。

# プレイテスト自動化とバランスチューニング記録 (2026-06-13)

## 目的
- ユーザーAIによるヘッドレス・プレイシミュレーション基盤を構築し、テストプレイを自動化する
- 一定フェーズ(WAVE13)から「死線」ハードモードに入り、生きるか死ぬかの難易度にする
- シミュレーション結果が合格基準を満たすまで調整をループする

## 基盤 (sim/)
- `harness.js` — index.html の `<script>` を Node vm + DOMスタブでロードし、描画なしで `update(dt)` を直接駆動。シード付き乱数(mulberry32)で決定的に再現
- `bot.js` — ユーザーAI 3ペルソナ + random ベースライン:
  - good: ミス3% / 反応5tick / 回避視野65px / 照準0.50
  - average: ミス14% / 反応6tick / 視野60px / 照準0.35 / AFK 0.4%/秒
  - weak: ミス18% / 反応9tick / 視野35px / 照準0.18 / AFK 1%/秒
  - 行動レイヤ: ゲート効用選択(捨てコストは軍規模比例) → ボス優先 → コミット帯域 → 局所回避+照準(小軍時は生存優先)
- `run.js` — 並列バッチランナー。時間上限600秒 + スコア上限200万(終わらないラン防止の二重保険)。
  `node sim/run.js --seeds 120 --seedBase 7777 --skills weak,average,good --cap 600 --out sim/results/x.json`

## 合格基準 (最終形)
- A 序盤公平性: 死線到達率 weak≥0.35 / average≥0.70 / good≥0.85
- B 致死性: average の死線後生存中央値 40〜150秒(±10%許容)、死線後240秒超の生存≤20%、cap/scoreCap到達≤10%
- C スキル表現: 到達率が単調かつ good−weak≥0.10、死線後中央値で weak最下位かつ good/weak≥1.3、総生存中央値 good≥average−5s かつ両者>weak
  - 注: 当初の「good≥average×1.15(総生存)」は、事前ハード216秒が全スキル共通の滑走路となる構造では数学的上限~1.10で達成不能と判明し、計測方法として上記に再定義した(2026-06-13)
- D 健全性: 同種罠ペア0 / NaN0 / エラー0 / 除染イベント≥ラン数÷5

## シミュレーションが駆動したゲーム変更 (時系列)
1. **序盤ランプ** — コールドスタート(初期5兵)は流入22.5HP/s vs DPS12.5で数学的に詰み。湧き間隔を開始時×2.5緩→90秒で通常化、初回ゲート1.5秒、初期10兵に
2. **ゲート弾の減衰貫通** — ゲートが弾を全吸収するとゲート下滞在中の防御力がゼロになり序盤死の主因に。+1付与後ダメージ半減で貫通に変更(「育てる間は火力半減」のトレードオフは維持)
3. **敵種アンロック遅延** — speeder lv3 / tank lv4 / splitter lv5 / shooter lv6 / diver lv7
4. **接触ロス上限** — HP由来項を「兵力18%+4」でキャップ(序盤タンク1接触全滅の理不尽除去。%ベースなので死線の致死性は維持)
5. **成長ガバナー** — 敵HP連動を外した結果、x2連鎖で兵数2.5億まで暴走(★システム崩壊)。進化コスト+兵数/40、3万兵超でx2の60%を+10%加算に置換 → 最終中央値 ~120万-320万
6. **事前ハードの群れ振替緩和** — count×1.5 / interval÷1.2(×2/÷1.5は事前ハードで湧き圧3倍となり、巨大軍でも%出血が収入を超えるコインフリップ化していた)
7. **死線=成長する洪水** — 巨大軍には敵HPは無意味(全ワンショット)で、実効脅威は「湧き数 vs 弾スループット(75発/秒上限)」のみと判明。定数上限を廃し、WAVE毎に count+0.25 / interval短縮+0.12 ずつ成長(上限4 / 2.4)。入口はwave12と連続、しかし必ず追い越してくる → 死亡時刻がスキル・軍規模で個別化
8. **HARD最終値**: startWave 13 / hpRamp 0.05 / countGrow 0.25 / intervalGrow 0.12 / eliteChance 0.08 / bossEvery 34 / volleyBonus 2

## ボット開発で判明した知見 (ゲーム設計へのフィードバック)
- ボス放置は巨大軍でも約-17%/秒の出血(8発×2.9秒の%弾幕)— ボス優先撃破が必須スキル。ボットにもボス優先行動を実装して初めて事前ハード到達が安定した
- 全帯域の最適回避位置スキャン(テレポート的挙動)は接触死を誘発 — 人間的な「局所回避」が正しい
- 「オーバーキル時はゲート下に常駐」戦略は照準レイヤの事前掃討効果を失い逆効果(実験して撤回)

## 既知のトレードオフ / 今後の候補
- 1ラン最大兵数の中央値は good で~320万(★32相当)。★オドメーターの希少性をさらに上げるなら governor 強化が候補だが、probe9 で確認した通り強すぎると事前ハード終盤の維持経済が崩れる(出血>収入)
- average の死線後中央値 38.3s は帯[40,150]の下端−4%(許容内)。もう数秒延ばすなら intervalGrow 0.12→0.10
- weak の到達率 0.74 は想定(0.35+)より高い。コールドスタートを更に厳しくする選択肢はあるが、初見体験を優先して現状維持
